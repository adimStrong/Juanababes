JuanBabes Analytics Dashboard - User Guide

What is JuanBabes Analytics?

A comprehensive dashboard for tracking Facebook page performance across 5 Juana Babe pages. Import data from Meta Business Suite, view engagement metrics, and analyze content performance.

Dashboard URL: https://juanababes-production.up.railway.app/

1. Getting Started

The JuanBabes Analytics Dashboard automatically tracks performance data from 5 Facebook pages. Data is imported from Meta Business Suite CSV exports and displayed in an interactive dashboard.

Access the Dashboard:
Open your browser and go to: https://juanababes-production.up.railway.app/

2. Dashboard Overview

The dashboard displays several key sections:

Top Statistics Cards

At the top of the dashboard, you'll see summary cards showing:

  • Total Posts - Number of posts across all pages
  • Total Views - Combined video/post views
  • Total Reach - Unique users who saw content
  • Total Engagement - Reactions + Comments + Shares
  • Total PES - Post Engagement Score (same as engagement)

Page Selector

Use the dropdown in the top-right corner to filter data by a specific page, or select "All Pages" to see combined data.

4. Understanding Metrics

Metric Description Why It Matters
Reactions Likes, loves, hahas, wows, sads, angrys Shows emotional response to content
Comments User replies on posts Indicates conversation and interest
Shares Users sharing content to their feed Best indicator of viral potential
Views Video/post view count Shows content visibility
Reach Unique users who saw the content True audience size
Engagement Reactions + Comments + Shares Overall interaction level
PES Post Engagement Score (= Engagement) Performance benchmark

5. Importing New Data

Step 1: Export from Meta Business Suite

1

Go to Meta Business Suite

Navigate to Insights > Content

2

Select date range (e.g., Dec 1-25, 2025)

Click Export and choose CSV format

3

Save the CSV file to:

C:\Users\us\Desktop\juanbabes_project\exports\from content manual Export\
Important: Meta exports timestamps in UTC. The system automatically converts to Philippine Time (PHT = UTC+8) during import.

6. Updating the Report

One-Click Update

The easiest way to update the report is to run the batch file:

1

Double-click update_report.bat in the project folder

2

Wait for the script to complete (about 30 seconds)

It will: Import CSVs → Export analytics → Push to Railway

3

Railway auto-deploys in 1-2 minutes

Refresh the dashboard to see updated data

Pro Tip: The dashboard shows data up to "today - 2 days" to ensure metrics are complete. Today and yesterday's data are excluded as they may be incomplete.

Manual Commands (Advanced)

# Import all CSVs
python csv_importer.py import-all "exports/from content manual Export" --mode merge

# Export analytics data
python export_static_data.py

# Push to Railway
git add -A
git commit -m "Update report"
git push origin main

8. Troubleshooting

Data not showing on dashboard?

  1. Check that CSV files exist in exports/from content manual Export/
  2. Run python csv_importer.py stats to verify database has data
  3. Run python export_static_data.py to regenerate analytics.json
  4. Push changes to GitHub and wait for Railway deploy

Dates look wrong?

Meta exports in UTC. The system should convert to PHT (+8 hours).

If dates are still wrong, re-import all CSVs:

python csv_importer.py import-all "exports/from content manual Export" --mode merge

Charts are empty?

Check the analytics.json file has data:

python -c "import json; d=json.load(open('frontend/public/data/analytics.json')); print('Posts:', len(d.get('posts',[]))); print('Daily:', len(d.get('daily',{}).get('all',[])))"

Git push failed?

  1. Check internet connection
  2. Verify GitHub credentials: git config --global user.name
  3. Try again: git push origin main
